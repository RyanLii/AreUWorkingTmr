## 1. 产品初心（你反复强调的最高优先级）
1. 这是一个“今晚贴心助手”（sweet companion），不是控酒/戒酒产品。
2. 不做说教，不做羞辱，不做“你这个月喝太多/花太多”的负面统计导向。
3. 只关心用户“今晚到明早”这一个窗口：
- 今晚如何更安全回家
- 明天如何更舒服、更少事故、能正常上班（如需）
4. 用户决定要喝酒后，App 帮他把过程做得更安全、更有把握。

## 2. 命名与品牌语气要求
1. 名字要风趣、有梗、轻松（例如 `Are U Working Tmr?` 这种风格）。
2. 语气要求：风趣但克制（funny but controlled），不轻浮、不过度严肃。
3. 文案必须像“关心你的朋友”，不是“审判你的教练”。

## 3. 核心定位（你确认过）
1. 核心功能不是“少喝”，而是“开始喝以后，帮你安全收场”。
2. 关键价值：
- 更准确记录今晚饮酒
- 给出可理解的恢复/可驾驶时间建议
- 到家后的恢复提醒（补水、电解质）

## 4. 产品形态与平台优先级
1. `watch-first` 是明确优先级（第一版本重点在 Apple Watch）。
2. iPhone 仍然需要（历史、配置、权限、后台定位、复杂展示）。
3. 你问过“是否可纯 Watch 运行”：
- 可以做很大一部分，但后台定位与某些持续能力通常由 iPhone 更稳。
4. 你最终要求：
- Watch：逻辑可以很复杂，但操作必须尽量简单。
- iPhone：UI 可以更复杂，逻辑跟随 Watch。

## 5. 核心用户流程（你定义的主链路）
1. 用户进酒吧后打开 App。
2. 点饮品类型（beer/wine/shot/cocktail/spirits/custom）。
3. 每次新买一杯就快速追加记录。
4. App 持续估算：
- standard drink（内部用于计算）
- BAC/状态
- 预计何时恢复到可驾驶阈值（按地区）
5. 用户离场/到家后收到恢复提醒。
6. 用户可点 `I'm Done Tonight` 结束本晚。

## 6. Watch 端详细要求（非常关键）
1. Beer 进入后要可选 size（如 schooner/pint 等）+ ABV。
2. 其他类别也要同等扩展：
- Wine：small/standard/large/或 ml
- Shot/Cocktail/Spirits/Custom：都要支持精细记录
3. 必须保留 `Use Default`（快速路径）。
4. 目标：记录尽量准确，让用户更可靠地估计恢复/开车时间。
5. 页面展示策略：
- 不要把“总共喝了多少 std”作为首页主要暴露
- 更应强调“从开始喝到现在，你何时恢复到目标状态/可驾驶阈值”
6. 当“尚未开始喝”时：
- 不显示状态卡
- 不显示 `I'm Done Tonight`
7. 你提过的交互心智：
- 用户很多时候已醉，信息要短、直接、能执行。

## 7. 文案与语气引擎要求（Done Tonight / 提醒）
1. `I'm Done Tonight` 按钮必须有趣。
2. 点击后输出有趣、关心、带一点幽默的短句。
3. 文案要按饮酒程度动态分层（tone by intoxication / std level）：
- 低量：温柔、诗意、轻松
- 中量：玩笑+收尾建议
- 高量：可更直接甚至“越界一点的玩笑”也可，但仍以安全为导向
4. 文案核心方向：
- “今晚已经很开心了，收好尾”
- “记得补水、回家、休息”
- 不做月度行为批评

## 8. Location / Reminder 触发要求
1. 漏记提醒：
- 同地点停留一段时间后发生移动
- 且一段时间无新增饮酒记录
- 触发“你是不是忘记记刚才那杯”
2. 到家提醒：
- 到家后延时一次提醒（仅一次）
- 提醒补水/电解质，帮助第二天状态
3. 你明确移除：
- `Car-Later Reminder`（次日取车提醒）需求不大，不做

## 9. 估算表达与法律风险要求
1. 你发现过“back to normal”不准确/不直观，要求改为更好懂表达：
- 类似“什么时候可以开车？”
2. 但必须明确：
- 仅建议，不构成法律依据
- 防止成为法律责任后门
3. 地区化要生效（你在澳洲，阈值要匹配当地标准）。
4. 同时保留你哲学：
- 即使用户不打算开车，也可把该时间当“恢复正常”的参考。

## 10. UI/视觉方向（多轮调整后的最终倾向）
1. 你曾要求“更夸张、日式元素”。
2. 之后又明确：
- Watch 端 UI 简洁，方便操作
- 重点在逻辑完整度和记录准确度
3. iPhone 端可承担更复杂视觉与信息展示。
4. 你同意过使用 `figma + figma-implement-design` 流程迭代。

## 11. 数据、权限、同步与默认
1. V1 语言：英文优先（你此前计划默认英文）。
2. 单位默认公制，可切换。
3. iPhone 负责后台定位触发；Watch 负责前台快速输入。
4. HealthKit 拒绝后可手动填写，不阻断核心功能。
5. 定位拒绝后仅降级智能提醒，核心记录与 ETA 保留。
6. 本地存储 + iCloud/CloudKit 同步（长期保留 + 手动清空）。

## 12. 1.1（你明确放入后续版本）
1. 酒类 icon 自定义（按国家/地区提供 top 20 风格选择）。
2. 首先覆盖 beer/wine 等主要入口 icon 体系。

## 13. 差异化目标（你要求与市面统计类工具拉开）
1. 不只是“事后统计”，而是“夜间实时助手”。
2. 强调“过程陪伴”：漏记补录、动态 ETA、到家恢复闭环。
3. 用户感受优先：
- 关心今晚体验与明早状态
- 而非长期道德化管理

## 14. 协作方式偏好（你明确要求过）
1. 尽量连续执行，不要频繁等确认。
2. 如遇可选项，默认选推荐方案，最后汇总给你看。
3. 你接受“先做再看结果”的推进方式。
